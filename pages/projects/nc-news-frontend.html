NC News Front-end

This project is the front-end of the NC News application, built during my Northcoders coding bootcamp. The application connects to the NC News API and renders dynamic data, allowing users to browse, read, and interact with articles and comments. A key focus was ensuring consistent behaviour across different states, including loading, error, and empty data scenarios.

I prioritised accurate data rendering, input validation, and predictable user interactions. Particular attention was given to how the interface responds to API delays, failed requests, and edge cases, ensuring users receive clear feedback at all times.

Testing and Behaviour Focus

  • Handled loading, error, and empty states clearly
  • Validated user input before submitting data to the API
  • Ensured consistent rendering when data updates dynamically
  • Verified voting and comment functionality under different conditions
  • Manually tested across screen sizes and devices for layout stability

Features

  • Browse articles with filtering by date or votes
  • Read full articles with structured content and comments
  • Post, delete, and vote on comments with immediate visual feedback
  • Upvote and downvote functionality for articles and comments
  • Responsive design across desktop and mobile devices
  • Integration with the NC News API

Challenges and Learnings

This project strengthened my understanding of how front-end behaviour must align with back-end logic. I learned to anticipate failure scenarios, test edge cases, and design interfaces that remain stable even when data is delayed, missing, or invalid. It reinforced the importance of clear feedback, predictable state changes, and defensive programming.
