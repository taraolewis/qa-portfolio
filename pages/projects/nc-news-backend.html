NC News API Back-end

This project is the API for the NC News application, built during my Northcoders coding bootcamp. I developed the endpoints and validated the behaviour using a test first workflow, writing automated tests to confirm correct responses, status codes, and error messages for both expected and unexpected requests. The API supports fetching and interacting with articles, topics, comments, and users, with strong input validation and clear error handling to ensure reliable and predictable behaviour.

Testing Focus

  • Unit and integration testing using Jest and Supertest
  • Coverage of success paths, invalid input, and missing resource cases
  • Validation of status codes, response shapes, and error messages
  • Manual checks using an API client to verify real request behaviour

Features

  • REST API for articles, comments, topics, and users
  • CRUD operations for comments
  • Sorting and filtering of articles by topic, author, and date
  • Error handling with descriptive messages for invalid requests
  • Hosted live version accessible via Render

Challenges and Learnings

Building this API strengthened my ability to think in terms of system behaviour and risk. I learned how to design tests that protect critical functionality, identify edge cases early, and support confident changes without breaking existing endpoints.

Tech Used

Node.js Express PostgreSQL Jest Supertest REST API Render
