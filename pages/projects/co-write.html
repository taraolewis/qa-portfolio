Co-Write: Collaborative Story Writing App

Co-Write is a mobile application developed as a group project during Northcoders Bootcamp. The application allows users to create and contribute to shared stories in real time, with authentication, data persistence, and external API integration. A key focus was ensuring reliable collaborative behaviour and consistent state management across multiple users.

Built over three weeks in an Agile team, the project required careful handling of user authentication, asynchronous updates, and API responses. I contributed to validating feature behaviour, testing edge cases, and debugging issues to maintain stable functionality across the application.

Testing and Behaviour Focus

  • Validated authentication flow using Firebase
  • Tested real time updates to ensure consistent shared state
  • Verified word limit enforcement and voting logic
  • Handled API delays and failure responses gracefully
  • Tested navigation flows across multiple mobile screens
  • Manually tested on different device sizes for layout stability

Features

  • User authentication with Firebase
  • Create and contribute to collaborative story threads
  • AI generated story continuation using external APIs
  • View completed stories with generated images
  • Real time updates to support collaborative editing
  • Word limit enforcement and voting system
  • Mobile first design built with React Native

Challenges and Learnings

This project strengthened my understanding of how collaborative systems must manage shared state and asynchronous behaviour. I learned to anticipate race conditions, inconsistent updates, and API failures, and to design defensive logic that protects the user experience.

Working in a team also reinforced the importance of clear communication, documenting issues, and testing new features carefully before merging changes.
