Mini Design System

A small component system created in Figma to define reusable interface patterns and clearly documented component states. The project emphasises consistency, predictable behaviour, and structured documentation to support reliable implementation and easier validation during development.

Testing and Behaviour Focus

  • Defined component states including default, hover, focus, error, and disabled
  • Considered validation messaging and error visibility
  • Established consistent spacing and typography rules to reduce UI defects
  • Documented expected behaviour to support clearer developer handoff
  • Designed components with accessibility and clarity in mind

Features

  • Core UI components including buttons, inputs, dropdowns, and modals
  • Shared typography, colour, and spacing foundations
  • Clear component naming and organisation
  • Basic usage guidelines and documentation

Challenges and Learnings

Creating the system strengthened my understanding of how clearly defined component states and documentation reduce ambiguity and help prevent inconsistent implementation. It reinforced the importance of defining expected behaviour early to support quality assurance and smoother collaboration between design and development.
